Comments
Java programs have three different types of comments:



  • End-of-line and multi-line comments are written primarily for yourself. They help you remember specific details about your source code.
  • Documentation comments, on the other hand, are written for others. They explain how to use your classes and methods in other programs.
    • Javadocs
    Javadoc is a documentation generator for the Java language for generating API documentation in HTML format from Java source code. The "doc comments" used by Javadoc is the de facto industry standard for documenting Java classes. Javadoc does not affect performance in Java as all comments are removed at compilation time.

    Method Signatures
    Method Overloading
    Within one class, you can have two (or more) definitions of a single method name. This is called overloading the method name.

    Method Overloading Rules
    When you overload a method name, any two definitions of the method name must have either different signatures, either;

    Different numbers of parameters
    OR
    Parameters must be of differing types
    OR
    The same types of parameters in a different order (ie int float, float int)
    Method Overloading Example
    
public class MyClass { 
  
    // Overloaded sum(). This sum takes two int parameters 
    public int sum(int x, int y) 
    { 
        return (x + y); 
    } 
  
    // Overloaded sum(). This sum takes three int parameters 
    public int sum(int x, int y, int z) 
    { 
        return (x + y + z); 
    } 
  
    // Overloaded sum(). This sum takes two double parameters 
    public double sum(double x, double y) 
    { 
        return (x + y); 
    }  
}

    Method Overloading Your turn...
    Are these overloaded methods valid?

    
int mymethod(int a, int b, float c)
int mymethod(int var1, int var2, float var3)
    Result: Compile time error. Argument lists are exactly same. Both methods are having same number, data types and same sequence of data types. 
    
int mymethod(int a, int b)
int mymethod(float var1, float var2)
    Result: valid, as there are different types of parameters 

    
int mymethod(int a, int b)
int mymethod(int num)
    Result: valid, as there are different number of parameters 
    
float mymethod(int a, float b)
float mymethod(float var1, int var2)
    Result: Perfectly fine. Valid case of overloading. Sequence of the data types of parameters are different, first method is having (int, float) and second is having (float, int). 

    
int mymethod(int a, int b)
float mymethod(int var1, int var2)
    Result: Compile time error. Argument lists are exactly same. Even though return type of methods are different, it is not a valid case. Since return type of method doesn’t matter while overloading a method. 
    
float mymethod(String a, float b)
float mymethod(float var1, int varB)
    Result: Perfectly fine. Valid case of overloading. Sequence of the data types of parameters are different, first method is having (String, float) and second is having (float, int).
    Constructor Overloading
    In Java, a constructor looks similar to a method but has no return type. Constructors can also be overloaded like Java methods. An overloaded constructor provides multiple ways to set up a new object. Like method overloading, the overloaded constructors differ by the number, types and sequence of the parameters passed to them. Remember a constructor with no parameter is the default constructor 
    
public class Demo
{
	private String someString;

	public Demo()
	{
		this.someString = "1";
	}
	public Demo(int Input)
	{
		this.someString = Integer.toString(Input);
	}
	public Demo(String Input)
	{
		this.someString= Input;
	}
	public Demo(Double Input)
	{
		this.someString= Double.toString(Input);
	}
}
    Constructor Gotcha!
    When we don’t implement any constructor, the java compiler inserts the default constructor into our code during compilation, however if we implement any constructor then compiler doesn’t generate a default constructor, which can cause errors if a no argument object is created. 
    
public class Demo
{
	private String someString;
	this.someString = "1";
	}
	public Demo(int Input)
	{
		this.someString = Integer.toString(Input);
	}
  
	public static void main(String args[])
	{
		//This statement would invoke no-arg constructor
		Demo obj = new Demo();
	}
}

    Control Flow Statements Review
    The statements inside your source files are generally executed from top to bottom, in the order that they appear.
    Control flow statements, however, break up the flow of execution by employing decision making, looping, and branching, enabling your program to conditionally execute particular blocks of code.
    if-then Statement
    It tells your program to execute a certain section of code only if a particular test evaluates to true.
    
String emotion = "happy";
if(emotion.equalsIgnoreCase("happy")
{
	System.out.println("I am Happy");
}
    if-then-else Statement
    The if-then-else statement provides a secondary path of execution when an "if" clause evaluates to false.
    
emotion = "sad";
if(emotion.equalsIgnoreCase("happy"))
{
	System.out.println("I am Happy");
}
else
{
	System.out.println("I am Sad");
}
    else-if Statement
    Use the else if statement to specify a new condition if the first condition is false
    
emotion = "excited";
if(emotion.equalsIgnoreCase("happy"))
{
	System.out.println("I am Happy");
}
else if(emotion.equalsIgnoreCase("excited"))
{
	System.out.println("I am Excited");
} 
else
{
	System.out.println("I am Sad");
}
    Switch Statement
    Unlike if-then and if-then-else statements, the switch statement can have a number of possible execution paths. A switch works with the byte, short, char, and int primitive data types. It also works with enumerated types, the String class, and a few special classes that wrap certain primitive types: Character, Byte, Short, and Integer.
     
    
String trafficLightColour = "yellow";
String actionString;        
switch (trafficLightColour) {
	case "red":  actionString = "stop";
			 break;
	case "yellow":  actionString = "slow";
			 break;
	case "green":  actionString = "go";
			 break;
	default: actionString = "Invalid traffic light colour";
			 break;
}
    Ternary operator
    Is a condensed form of the if-else statement that also returns a value.
    The ternary operator ?: in Java is the only operator which accepts three operands:
    variable = booleanExpression ? expression1 : expression2
    The very first operand must be a boolean expression, the second and the third operands can be any expression that returns some value.
    The ternary construct returns expression1 as an output if the first operand evaluates to true, expression2 otherwise. 
    
Double number = -5.5;
String result;

result = (number > 0.0) ? "positive" : "not positive";
System.out.println(number + " is " + result);
    while loop
    The while loop loops through a block of code as long as a specified condition (boolean) is true.
    
int count = 1;
while (count < 6) //specified condition, evaluates to false when count = 6
{
	System.out.println("Count is: " + count);
	count++;
}
//with break & continue
count = 1;
while (true) {
	if(count > 5)
	{
		System.out.println("Break, breaks out of the loop");
		break;
	}
	if(count == 3)
	{
		count++;
		System.out.println("Continue starts the loop again");
		continue;
	}
	System.out.println("Count is: " + count);
	count++;
}
    do-while loop
    This loop will execute the code block once, before checking if the condition is true, then it will repeat the loop as long as the condition is true. 
    
int numeral = 1;
do{
	System.out.println("Count is: " + numeral);
	numeral++;            
}while(numeral < 6);
    for loop
    Running the same code multiple times is called iteration.
    This type of loop is so common that there is another statement, the for loop, that expresses it more concisely.     
    
for (int i = 2; i <= 8; i += 2) 
{
    System.out.print(i + ", ");
}
System.out.println("Who do we appreciate?");
    For loops have three components in parentheses, separated by semicolons:
    the initializer, the condition, and the update.
    1.The initializer runs once at the very beginning of the loop. (It is equivalent to the line before the while statement.)
    2.The condition is checked each time through the loop. If it is false, the loop ends. Otherwise, the body of the loop is executed (again).
    3.At the end of each iteration, the update runs, and we go back to step 2.
    for-each loop

String[] cars = {"Volvo", "BMW", "Ford", "Mazda"};
//syntax (Variable type in array variable name : array name)
for (String c : cars) 
{
  System.out.println(c);

    Nested Loops
    Like conditional statements, loops can be nested one inside the other. Nested loops allow you to iterate over two variables. For example, we can generate a “multiplication table” like this: 
    
for (int x = 1; x <= 10; x++) {
    for (int y = 1; y <= 10; y++) {
        System.out.printf("%4d", x * y);
    }
    System.out.println();
}
    Each loop repeats their corresponding statements 10 times. The outer loop iterates from 1 to 10 only once, but the inner loop iterates from 1 to 10 each of those 10 times. As a result, the printf method is invoked 100 times.
