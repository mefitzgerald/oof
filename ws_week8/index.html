Comments
Java programs have three different types of comments:



  • End-of-line and multi-line comments are written primarily for yourself. They help you remember specific details about your source code.
  • Documentation comments, on the other hand, are written for others. They explain how to use your classes and methods in other programs.
    • Javadocs
    Javadoc is a documentation generator for the Java language for generating API documentation in HTML format from Java source code. The "doc comments" used by Javadoc is the de facto industry standard for documenting Java classes. Javadoc does not affect performance in Java as all comments are removed at compilation time.
    Javadoc description

    Method Signatures
    Method Overloading
    Within one class, you can have two (or more) definitions of a single method name. This is called overloading the method name.

    Method Overloading Rules
    When you overload a method name, any two definitions of the method name must have either different signatures, either;

    Different numbers of parameters
    OR
    Parameters must be of differing types
    OR
    The same types of parameters in a different order (ie int float, float int)
    Method Overloading Example
    
public class MyClass { 
  
    // Overloaded sum(). This sum takes two int parameters 
    public int sum(int x, int y) 
    { 
        return (x + y); 
    } 
  
    // Overloaded sum(). This sum takes three int parameters 
    public int sum(int x, int y, int z) 
    { 
        return (x + y + z); 
    } 
  
    // Overloaded sum(). This sum takes two double parameters 
    public double sum(double x, double y) 
    { 
        return (x + y); 
    }  
}

    Method Overloading Your turn...
    Are these overloaded methods valid?

    
int mymethod(int a, int b, float c)
int mymethod(int var1, int var2, float var3)
    Result: Compile time error. Argument lists are exactly same. Both methods are having same number, data types and same sequence of data types. 
    
int mymethod(int a, int b)
int mymethod(float var1, float var2)
    Result: valid, as there are different types of parameters 

    
int mymethod(int a, int b)
int mymethod(int num)
    Result: valid, as there are different number of parameters 
    
float mymethod(int a, float b)
float mymethod(float var1, int var2)
    Result: Perfectly fine. Valid case of overloading. Sequence of the data types of parameters are different, first method is having (int, float) and second is having (float, int). 

    
int mymethod(int a, int b)
float mymethod(int var1, int var2)
    Result: Compile time error. Argument lists are exactly same. Even though return type of methods are different, it is not a valid case. Since return type of method doesn’t matter while overloading a method. 
    
float mymethod(String a, float b)
float mymethod(float var1, int varB)
    Result: Perfectly fine. Valid case of overloading. Sequence of the data types of parameters are different, first method is having (String, float) and second is having (float, int).
    Constructor Overloading
    In Java, a constructor looks similar to a method but has no return type. Constructors can also be overloaded like Java methods. An overloaded constructor provides multiple ways to set up a new object. Like method overloading, the overloaded constructors differ by the number, types and sequence of the parameters passed to them. Remember a constructor with no parameter is the default constructor 
    
public class Demo
{
	private String someString;

	public Demo()
	{
		this.someString = "1";
	}
	public Demo(int Input)
	{
		this.someString = Integer.toString(Input);
	}
	public Demo(String Input)
	{
		this.someString= Input;
	}
	public Demo(Double Input)
	{
		this.someString= Double.toString(Input);
	}
}
    Constructor Gotcha!
    When we don’t implement any constructor, the java compiler inserts the default constructor into our code during compilation, however if we implement any constructor then compiler doesn’t generate a default constructor, which can cause errors if a no argument object is created. 
    
public class Demo
{
	private String someString;
	this.someString = "1";
	}
	public Demo(int Input)
	{
		this.someString = Integer.toString(Input);
	}
  
	public static void main(String args[])
	{
		//This statement would invoke no-arg constructor
		Demo obj = new Demo();
	}
}
